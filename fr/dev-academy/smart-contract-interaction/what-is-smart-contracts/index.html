Que sont les contrats intelligents ?

Wikipedia Un contrat intelligent est un programme informatique ou un protocole de transaction destiné à exécuter, contrôler ou documenter automatiquement des événements et des actions juridiquement pertinents selon les termes d'un contrat ou d'un accord. Les objectifs des contrats intelligents sont la réduction du besoin d'intermédiaires de confiance, des coûts d'arbitrage et d'exécution, des pertes dues à la fraude, ainsi que la réduction des exceptions malveillantes et accidentelles. :::

En bref, les contrats intelligents sont des logiques exécutables sur une chaîne. L'exécution en chaîne et décentralisée rend la logique imparable, incassable et irréversible. Cela rend l'exécution fiable et donc la validité sécurisée par la cryptographie.

Les contrats intelligents peuvent représenter une logique généralisée allant des applications financières aux structures organisationnelles.

Dans notre contexte, CosmWasm est le cadre qui fournit le code et les outils pour développer des contrats intelligents et l'infrastructure pour les exécuter sur la chaîne. Pour l'instant, les contrats intelligents sont écrits en utilisant Rust.

Voici un court code montrant la logique d'exécution du contrat intelligent pour aider à la compréhension.

/// Cette logique est utilisée pour transférer des jetons/espèces d'un compte à un autre
pub fn execute_transfer(
    deps : DepsMut,
    _env : Env,
    info : MessageInfo,
    recipient : String,
    amount : Uint128,
) -> Result<Response, ContractError> {
    // voici une validation du montant
    if amount == Uint128::zero() {
        return Err(ContractError::InvalidZeroAmount {}) ;
    }

    let rcpt_addr = deps.api.addr_validate(&recipient)? ;

    // soldes mis à jour dans le magasin
    BALANCES.update(
        deps.storage,
        &info.sender,
        |balance : Option<Uint128>| -> StdResult<_> {
            Ok(balance.unwrap_ou_default().checked_sub(amount) ?)
        },
    )? ;
    BALANCES.update(
        deps.storage,
        &rcpt_addr,
        |balance : Option<Uint128>| -> StdResult<_> { Ok(balance.unwrap_ou_default() amount) },
    )? ;

    // réponse renvoyée pour indiquer que l'exécution est réussie et poursuivre l'exécution
    let res = Response::new()
        .add_attribute("action", "transfer")
        .add_attribute("from", info.sender)
        .add_attribute("to", recipient)
        .add_attribute("amount", amount) ;
    Ok(res)
}

Comme vous pouvez le constater, les contrats intelligents ne sont pas magiques. Ils sont très similaires à toute autre logique programmée.

Les différences par rapport à tout développement de backend sont de nouveaux concepts issus de la cryptographie et d'autres disciplines :

  • hash
  • adresse
  • racine de merkle
  • base de données clé-valeur

Bien sûr, il existe d'autres concepts : le consensus, la sécurité du réseau, la propagation des messages . Mais il s'agit de sujets avancés qui ne seront pas abordés ici.
