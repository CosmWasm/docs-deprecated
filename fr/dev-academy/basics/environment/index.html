Configuration de l'environnement

Vous avez besoin d'un environnement pour exécuter les contrats. Vous pouvez soit exécuter votre nœud localement, soit vous connecter à un réseau existant. For easy testing, you can use the public testnet (cliffnet) to deploy and run your contracts.

When interacting with a network, you can either use wasmd which is a Go client or the Node REPL. Le Node REPL est recommandé pour les opérations contractuelles, car la manipulation de JSON n'est pas intuitive avec le client shell/Go.

Pour ce cours, nous utiliserons le testnet public pour simplifier les choses.

Gitpod​

Gitpod est un environnement de développement en ligne. Nous avons une image gitpod sur laquelle vous pouvez baser vos projets. L'image contient toutes les pré-requis ci-dessous. Gitpod est recommandé si vous disposez d'une connexion Internet stable. Ajoutez le fichier .gitpod.yml à la racine de votre projet puis poussez-le sur GitHub. After installing gitpod extension, on the github project repo, there will be a Gitpod button which will create a workspace for you to work on.

Go​

You can setup golang following the official documentation. Les dernières versions de wasmd require go version 1.16.8+.

Rust​

En supposant que vous n'avez jamais travaillé avec Rust, vous devrez d'abord installer des outils. L'approche standard consiste à utiliser rustup pour télécharger les dépendances et gérer la mise à jour de plusieurs versions de cargo et rustc, que vous utiliserez.

Installer Rust sous Linux et Mac​

Tout d'abord, installer rustup. Une fois installé, assurez-vous que vous avez la target wasm32 :

rustup default stable
cargo version
# If this is lower than 1.55.0+, update
rustup update stable

rustup target list --installed
rustup target add wasm32-unknown-unknown

wasmd​

wasmd is the backbone of the CosmWasm platform. It is an implementation of a Cosmoszone with wasm smart contracts enabled.

This code was forked from the cosmos/gaia repository as a base and then x/wasm was added and many gaia-specific files were cleaned up. Cependant, le binaire wasmd devrait fonctionner comme gaiad, à l'exception de l'ajout du module x/wasm. .

Si vous avez l'intention de développer ou d'éditer un contrat, vous avez besoin de wasmd.

git clone https://github.com/CosmWasm/wasmd.git
cd wasmd
# replace the v0.23.0 with the most stable version on https://github.com/CosmWasm/wasmd/releases
git checkout v0.23.0
make install

# verify the installation
wasmd version
Si vous avez des problèmes ici, vérifiez votre PATH. make install copiera wasmd dans $HOME/go/bin par défaut, assurez-vous que cela est configuré dans votre PATH également, ce qui devrait être le cas en général pour compiler du code Go à partir des sources. :::

Configurer wasmd et Wallet​

Let's configure wasmd exec, point it to testnets, create a wallet and ask for tokens from the faucet:

First source the cliffnet cosmwasm public network configurations to the shell:

source <(curl -sSL https://raw.githubusercontent.com/CosmWasm/testnets/master/cliffnet-1/defaults.env)

Configurer le client :

# ajouter des portefeuilles pour les tests
wasmd keys add wallet
>

  "name" : "wallet",
  "type" : "local",
  "address" : "wasm13nt9rxj7v2ly096hm8qsyfjzg5pr7vn5saqd50",

  "mnemonic" : "hobby bunker rotate piano satoshi planet network verify else market spring toward pledge turkey tip slim word jaguar congress thumb flag project chalk inspire"
}

Vous devez avoir des jetons dans votre adresse pour interagir. If you are using a local node you can skip this step. Requesting tokens from the faucet:

JSON=$(jq -n --arg addr $(wasmd keys show -a wallet) '{"denom":"upebble","address":$addr}') && curl -X POST --header "Content-Type: application/json" --data "$JSON" https://faucet.cliffnet.cosmwasm.com/credit

Exporter les Paramètres de wasmd​

wasmd client requires setup for interacting with different testnets. Chaque réseau de test possède ses propres endpoints et paramètres système.

La meilleure façon de configurer wasmd est de configurer les variables d'environnement ci-dessous :

# bash
export NODE="--node $RPC"
export TXFLAG="${NODE} --chain-id ${CHAIN_ID} --gas-prices 0.025upebble --gas auto --gas-adjustment 1.3"

ou

# zsh
export NODE=(--node $RPC)
export TXFLAG=($NODE --chain-id $CHAIN_ID --gas-prices 0.001upebble --gas auto --gas-adjustment 1.3)

If the command above throws an error, this means your shell is different. If there are no errors, try running this:

wasmd query bank total $NODE

Cela signifie que vous pouvez maintenant interagir avec le nœud que vous avez configuré. Vous pouvez vérifier que votre demande au robinet a abouti en vérifiant la balance de votre portefeuille en essayant la commande :

wasmd query bank balances $(wasmd keys show -a wallet) $NODE

et vous pouvez regarder les différentes commandes en explorant

wasmd help

Configuration des outils linux​

We will be using a few linux tools extensively:

apt install jq curl

Configuration du client JS CLI​

Une autre façon d'utiliser et d'interagir avec les contrats on-chain est le client interactif CosmJS @cosmjs/cli.

Pour l'utiliser, installez node.js 12+ et npx.

npx @cosmjs/cli@^0.26 --init https://raw.githubusercontent.com/InterWasm/cw-plus-helpers/main/base.ts --init https://raw.githubusercontent.com/InterWasm/cw-plus-helpers/main/cw20-base.ts

Now you will see that an interactive shell popped up.

Code below sets up a client that speaks to cliffnet, generates an address and then requests tokens from the faucet. "password" est le mot de passe du fichier key. This key is different from the wasmd key generated above

const [addr, client] = await useOptions(cliffnetOptions).setup("password");
client.getAccount(addr);

Vous devriez voir quelque chose qui ressemble à :

{
  adresse : 'wasm1kfaqnxcsz6pwxyv0h468594g6g2drwxfrrwslv',
  pubkey : null,
  accountNumber : 326,
  sequence : 0
}
