Les bases de Cosmos SDK et CosmWasm

Le Cosmos SDK​

From Cosmos SDK docs

info

Qu'est-ce que le SDK ? Le Cosmos-SDK est un framework open-source permettant de construire des blockchains publiques de type PoS (Proof-of-Stake) multi-actifs, comme le Cosmos Hub, ainsi que des blockchains de type PoA (Proof-Of-Authority) autorisées. Les blockchains construites avec le kit SDK Cosmos sont généralement appelées blockchains spécifiques à une application (application-specific). L'objectif du SDK Cosmos est de permettre aux développeurs de créer facilement des blockchains personnalisées à partir de zéro, qui peuvent nativement interagir avec d'autres blockchains. Nous envisageons le SDK comme un framework semblable à npm afin de construire des applications blockchain sécurisées au-dessus de Tendermint. Les blockchains basées sur un SDK sont construites à partir de modules composables, dont la plupart sont open source et facilement accessibles à tout développeur. Tout le monde peut créer un module pour le Cosmos-SDK, et l'intégration de modules déjà construits est aussi simple que leur importation dans votre application blockchain. De plus, le SDK Cosmos est un système basé sur les capacités, ce qui permet aux développeurs de mieux raisonner sur la sécurité des interactions entre les modules.

In short, Cosmos SDK is an easy-to-use blockchain framework that enables developers to build application-specific blockchains without delving into the implementation details of low-level blockchain infrastructure (e.g., Byzantine fault tolerance).

You can read through Cosmos SDK documentation for more information:

  • Cosmos SDK Intro gives a great introduction to the framework.
  • Cosmos SDK Basics covers the basics of Cosmos SDK concepts that are required for developing on CosmWasm with minimal Cosmos SDK knowledge.

CosmWasm​

CosmWasm est écrit comme un module qui peut s'insérer dans le SDK Cosmos. Cela signifie que toute personne qui construit actuellement une blockchain en utilisant le SDK Cosmos peut rapidement et facilement ajouter le support des smart contracts CosmWasm à sa chaîne, sans ajuster la logique existante. Nous fournissons également un exemple de binaire de CosmWasm intégré dans le binaire gaiad, appelé wasmd, afin que vous puissiez lancer une nouvelle blockchain compatible avec les smart contracts, en utilisant des outils documentés et testés et le même modèle de sécurité que le Cosmos Hub. :::

En bref, CosmWasm est un module de contrat intelligent (smart contract) qui peut être utilisé avec Cosmos SDK. Il exécute la machine virtuelle Web Assembly (ou Wasm), permettant aux développeurs de créer des contrats intelligents dans différents langages. Actuellement, il prend en charge les contrats intelligents écrits en Rust.

The Cosmos SDK is built on top of Tendermint Core, which handles the low-level infrastructure of maintaining a blockchain (e.g., transacting, consensus). Il interagit avec le Cosmos SDK via l'ABCI, ou Application BlockChain Interface. Les nœuds complets de la blockchain exécutent toute cette "stack" de Cosmos SDK et Tendermint pour répliquer la blockchain et valider les transactions.

Les transactions sont transmises de l'ABCI à l'application, qui les achemine à son tour vers des modules chargés d'exécuter la logique applicative. CosmWasm est un module qui exécute la machine virtuelle WebAssembly.

Par rapport aux contrats intelligents CosmWasm, les modules natifs de Cosmos SDK sont légèrement plus rapides en raison de l'absence de virtualisation. Cependant, la virtualisation comporte ses avantages propres. Smart contracts can be swapped when the chain is running, whereas restarts are required for native modules. De plus, grâce à la machine virtuelle flexible Wasm, les contrats CosmWasm peuvent être écrits en Rust (et un jour d'autres langages également).
